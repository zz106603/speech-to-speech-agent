# Project Rules

1. 모든 응답은 한국어로 작성한다.
2. 코드 작업 전 docs/ 폴더의 Markdown 문서를 확인한다.
3. 구현 전 관련 코드, 설정, 문서를 먼저 조사한다.
4. 모르는 구조는 추정하지 않는다.
5. 요구사항이 불명확하면 사용자에게 확인한다.
6. 요청 범위를 벗어난 파일은 수정하지 않는다.
7. 불필요한 리팩토링은 수행하지 않는다.
8. 요청되지 않은 기술 스택 변경은 수행하지 않는다.
9. 임시 우회 코드로 문제를 해결하지 않는다.
10. OpenAI Realtime 관련 구현은 공식 문서를 우선 확인한다.
11. WebRTC 연결 흐름(Offer/Answer, ICE, DataChannel)을 임의로 변경하지 않는다.
12. 로그인, 회원가입, DB, 멀티유저 기능을 추가하지 않는다.
13. 과제 요구사항 범위를 벗어난 기능을 구현하지 않는다.
14. React Native 작업은 Android 기준으로 진행한다.
15. 사용자가 요청하지 않은 경우 테스트 실행은 생략한다.
16. 작업 완료 후 아래 내용을 요약한다.
    * 변경 파일
    * 변경 내용
    * 변경 이유
    * 수행한 검증
17. 사용자가 커밋을 요청한 경우:
    - 관련 변경만 묶어서 커밋한다.
    - 커밋 메시지는 "type: 한글 설명" 형식을 사용한다.
예시:
feat: 실시간 토큰 발급 API 추가
fix: WebRTC 연결 오류 수정
docs: 페르소나 시트 작성
chore: 프로젝트 초기 구조 생성
18. 사용자가 push를 요청한 경우:
    - 현재 브랜치 기준으로 push 한다.
    - push가 필요한 이유가 명확하지 않으면 사용자에게 확인한다.
